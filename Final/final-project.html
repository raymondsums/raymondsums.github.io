01

Use the correct HTML element for your content

The website do not use proper headings to represent content hierarchy, some H2 headings are nested within H5 headings
Solution : Make H5 headings come after H2 headings, organize headings by content hierarchy instead of style. Style headings' font-size with CSS rather than HTML.

02

Support keyboard navigation

The content of the site represented by id="main-content" has a tabindex attribute of -1. As a result, user will tab through the page without the ability to interact with the content. There is also no way to interact with the button, represented by class="et_pb_toggle_title" because it is a pseudo-element, meaning users navigating the website with keyboard is not able to expand the accordion.
Solution: Make the website more "tab-able" by including tab-index values on the items inside id="main-content" and make the button to toggle accordion possible to be tabbed on, by creating them with html instead of css.

03

Give users a way to skip top level navigation to access main content

Create alternate routes for users to access information

Understand and use HTML landmarks

Solution: By making tabindex for id="main-content" equal to 1, we can allow users who use assistive device to be able to skip specific areas of the page to areas that they are interested in.

04

Write good alt text for your images

This is done well for the most part. There are texts present for the larger images/logo such as the 'Michigan Alumnus' and 'Michigan Alumni Association' logo. However, there are no alt texts present for li items in the main menu (id="menu-main-menu"), where the M block and open book icons are located. Similarly, the search icon (id="et_search_icon") is also missing alt text.
Solution: Include alt text for the font icons, such as the search icon
